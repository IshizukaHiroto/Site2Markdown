## 差別化の核心

> **「設定が多いのに、使い始めるのが一番早い。」**

競合（MarkDownload・Obsidian Web Clipper）はいずれも機能豊富だが、
設定が複雑すぎる・UIが古い・初回設定に手間がかかる、という共通の弱点を持つ。
Site2Markdown はポップアップを開いた瞬間に変換が完了している「即効性」と
洗練された UI で、すべてのユーザー層を迎え入れる。

## Persona 1 — AI Prompt Builder「渡辺 拓也」

**属性**
- 27歳・IT系スタートアップのPM
- ChatGPT / Claude を1日10回以上使う
- Webで調べた情報をそのままAIに渡す作業を毎日繰り返している

**典型的なシナリオ**
> 技術ブログを読んでいる途中でAIに「この記事を要約して」と聞きたくなる。
> URLを貼っても読んでくれないことが多いので、記事のテキストをコピーしたい。
> でも普通にCtrl+A → コピーすると広告やナビも全部入ってしまって汚い。

**求めているもの**
- ナビ・広告なしのクリーンなテキストを **1クリックで** 取得したい
- フロントマターは不要。純粋な本文だけほしい
- コピーして即貼り付けたい（保存は不要）

**Site2Markdownがどう解決するか**
- ポップアップを開いた瞬間に本文だけ抽出・変換完了
- 「コピー」ボタン1発でクリップボードへ
- `autoCopy` 設定をONにすれば開くだけで自動コピー（Pro機能）

**フリーミアム転換の動機**
- 自動コピーが欲しくなったとき → Pro へ
- 毎日使う人ほど「$9払ってでも自動化したい」と感じやすい

## Persona 2 — Second Brain「佐藤 みき」

**属性**
- 24歳・大学院生（研究者志望）
- Obsidian を使った PKM（Personal Knowledge Management）を実践中
- Zettelkasten 方式でノートをリンクさせて知識を蓄積している

**典型的なシナリオ**
> 論文・技術ブログ・ニュース記事を読んで、Obsidian の Vault にクリップしたい。
> Obsidian Web Clipper は Vault のパーミッション設定が面倒で毎回つまずく。
> MarkDownload は使っているが、フロントマターのカスタマイズができず、
> 毎回手動でメタデータを追記しないといけない。

**求めているもの**
- `title`, `url`, `date`, `tags` を自動でフロントマターに入れてほしい
- ファイル名を `YYYY-MM-DD_タイトル.md` 形式で保存したい
- テーブルやコードブロックが崩れないでほしい

**Site2Markdownがどう解決するか**
- フロントマター設定で `title` / `url` / `date` / `tags` をフル制御できる
- ファイル名フォーマットを設定画面で細かく指定可能
- Turndown + GFM プラグインでテーブル・コードブロックを正確に変換

**フリーミアム転換の動機**
- カスタムフロントマター（`source:`, `status: draft` 等）が使いたくなったとき
- このペルソナは最も課金意欲が高い。Obsidian 自体に $50/年払っている層

## Persona 3 — Developer「田中 健吾」

**属性**
- 29歳・バックエンドエンジニア
- GitHub / Zenn / 技術ドキュメントを毎日読む
- ローカルのメモを Markdown で管理している（VSCode + Git）

**典型的なシナリオ**
> 公式ドキュメントや Stack Overflow の回答を Markdown でメモしたい。
> コードブロックの言語指定が正確に変換されないツールは使い物にならない。
> 余計なリンクが大量に入るのも嫌で、テキストだけほしいことが多い。

**求めているもの**
- コードブロックの言語指定（` ```python ` 等）が正確に変換されること
- リンクを除去してテキストだけにするオプションがほしい
- 見出しレベルを調整したい（他の文書に貼るときにh1が重複する問題）

**Site2Markdownがどう解決するか**
- `linkHandling: textOnly` でURLをすべて除去できる
- `headingShift` で見出しを1〜2段下げられる
- GFM プラグインでコードブロックの言語指定を正確に保持

**フリーミアム転換の動機**
- `linkHandling` と `headingShift` は Pro 機能なので、使いたければ課金
- 「$9 なら即払う」という層。価格に鈍感なエンジニアが多い

## Persona 4 — Content Collector「山田 さら」

**属性**
- 26歳・マーケター / コンテンツクリエイター
- Notion をチームのナレッジベースとして使っている
- Web記事のリサーチをNotionにまとめる作業が多い

**典型的なシナリオ**
> 競合他社の記事や業界トレンドを Notion にクリップして整理したい。
> ブラウザの Notion Web Clipper は使っているが、
> Markdown ではなく Notion ブロック形式で入るため、
> フォーマットが崩れてチームに共有しにくいことがある。
> Markdown テキストとして取得してから Notion に貼れば綺麗に入る。

**求めているもの**
- 記事の本文だけをきれいに取り出したい
- 操作が簡単でないと社内に広められない（非エンジニアでも使える）
- ファビコン・サイト名が見えるとソース管理しやすい

**Site2Markdownがどう解決するか**
- ポップアップがシンプルで直感的。非エンジニアでも迷わない
- `#page-source` にファビコン・ドメイン名が表示される
- コピーしてそのまま Notion に貼れる

**フリーミアム転換の動機**
- 自動コピー（毎回ボタンを押さなくていい）
- チームで使う場合はチームメンバー全員分の購入 → 口コミ拡散につながる

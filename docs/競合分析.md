# 競合分析

作成: 2026-03-20

---

## 競合マップ

```
設定の豊富さ
        ↑
        │  Obsidian            Site2Markdown ★
        │  Web Clipper         （目指す位置）
        │
        │         MarkDownload
        │
        │  Roam               Copy as
        │  Highlighter        Markdown
        └──────────────────────────────────→
        UIがシンプル・使いやすい →
```

**Site2Markdown の勝ち筋:**
設定が豊富なのに、使い始めのハードルが最も低い。インストールして開けばすぐ使える。

## 競合詳細比較

### MarkDownload
- **価格:** 無料・オープンソース（寄付モデル）
- **評価:** 4.69/5（153件）
- **GitHub Stars:** 約3,600〜3,700
- **対応ブラウザ:** Chrome / Firefox / Edge / Safari

**強み:**
- 長年の実績があり、PKMコミュニティでの認知度が高い
- Firefox にも対応

**弱み:**
- UIが古く、直感的でない
- テーブル・コードブロックの変換精度が低い
- コピー機能が弱い（ダウンロードが主な操作）
- MV3 対応のため 2024〜2025年にかけて全面書き直し中。開発が停滞している
- Chrome Web Store のコンプライアンス問題で一時非掲載になった経緯あり

**Site2Markdown との差:**

| 比較項目 | MarkDownload | Site2Markdown |
|---------|-------------|---------------|
| UI・デザイン | 古い・無骨 | Editorial デザイン、洗練されている |
| コピー操作 | ダウンロードが主 | コピーボタンが中心、1クリック |
| フロントマター | 基本的な設定のみ | description / author / tags / カスタムフィールドまで対応 |
| ファイル名制御 | 限定的 | 日付形式・区切り文字・最大文字数を細かく設定可能 |
| 開発状況 | 停滞中（MV3移行で全面書き直し） | アクティブに開発中 |

---

### Obsidian Web Clipper（公式）
- **価格:** 無料・オープンソース（Obsidian 公式開発）
- **評価:** 4.6/5 前後（100,000+ 週次アクティブユーザー）
- **対応ブラウザ:** Chrome / Firefox / Safari / Edge

**強み:**
- Obsidian 公式製で、Vault への直接保存に最適化されている
- テンプレート機能が強力（変数・条件分岐が使える）
- オフライン動作・プライバシー重視

**弱み:**
- Vault のパーミッション設定が必要で、初回セットアップが煩雑
- テンプレート構文の学習コストが高く、非エンジニアには難しい
- Obsidian ユーザー以外には使いにくい（特定ツール依存）
- 画像のクリッピングが失敗することがある

**Site2Markdown との差:**

| 比較項目 | Obsidian Web Clipper | Site2Markdown |
|---------|---------------------|---------------|
| 対象ユーザー | Obsidian ユーザー専用 | Obsidian / Notion / AI など全方位 |
| 初期設定 | Vault 設定が必要（複数ステップ） | インストール → 即使える |
| テンプレート | 強力だが学習コストが高い | シンプルなフロントマター設定 |
| 特定ツール依存 | Obsidian 必須 | どこにでもペーストできる純粋な Markdown |

---

### Copy as Markdown
- **価格:** 無料
- **評価:** 3.5/5
- **対応ブラウザ:** Chrome / Firefox

**強み:**
- 右クリックメニューからリンク・画像・タブを Markdown 形式でコピーできる
- 操作が極めてシンプル

**弱み:**
- ページ全体の変換機能がない（リンクや選択テキストのコピー特化）
- Gmail / Notion などの特定サイトで動作しないことがある
- 評価が競合の中で最も低い

**Site2Markdown との差:** 用途が異なる（部分コピー vs. ページ全体の変換）。競合というより補完的なツール。

---

### Roam Highlighter
- **価格:** 無料（寄付モデル）
- **評価:** 4.62/5（55件）
- **ターゲット:** Roam Research / Obsidian / Logseq / Notion ユーザー

**強み:**
- テキストハイライト機能に特化
- PKM ツールへの出力フォーマットが整っている

**弱み:**
- Roam Research 自体が 2024年以降に顕著に衰退（サブRedditが2週間投稿ゼロになる事態も）
- 非ラテン文字（日本語等）の扱いが弱い
- PDF 未対応

**Site2Markdown との差:** ハイライト特化 vs. ページ全体変換。日本語対応の差も大きい。

---

### SingleFile
- **価格:** 無料・オープンソース
- **評価:** 4.36/5（1,038件）

**強み:**
- CSS・画像・フォントごとページを 1つの HTML ファイルに保存できる
- Google Drive / Dropbox / GitHub 連携対応

**弱み:**
- 出力が HTML であり、Markdown ではない
- 保存先フォルダが選択できない（Downloads フォルダ固定）

**Site2Markdown との差:** 目的が根本的に異なる（HTML保存 vs. Markdown変換）。競合関係はほぼない。

## 差別化サマリー

### 競合が誰も取り組んでいない軸

1. **日本語ユーザーへの正面からの対応**
   UI・説明文・サポートがすべて日本語。国内 PKM コミュニティ（Obsidian・Notion）への訴求が他の競合に比べて圧倒的に有利。

2. **「即使えるのに設定が豊富」の両立**
   - Obsidian Web Clipper: 設定は豊富だが初回セットアップが重い
   - MarkDownload: 設定はシンプルだが UI が古く機能も限定的
   - Site2Markdown: インストールしてすぐ使えて、深く使いたければ設定で細かく制御できる

3. **AI ユーザー向けの最適化**
   - `linkHandling: textOnly`（URL をすべて除去してノイズを減らす）
   - `collapseBlankLines`（トークン数を節約）
   - `autoCopy`（開くだけでクリップボードに入る）
   これらを組み合わせると「AIへの貼り付け専用ツール」として他に類がない存在になれる。

### 競合が狙っていない市場

- **日本国内 PKM ユーザー:** MarkDownload・Obsidian Web Clipper は日本語対応が薄い。Zenn / Qiita での技術記事発信と相性が良い
- **AI ヘビーユーザー:** 「Web記事 → AI プロンプト」というワークフローに特化した拡張機能はまだ存在しない

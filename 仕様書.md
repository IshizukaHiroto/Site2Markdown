# Site2Markdown 仕様書

作成: 2026-03-20

---

## 概要

任意のWebページをMarkdown形式に変換し、AIへの入力・Obsidian・Notionへのクリップ・ローカル保存に活用できるChrome拡張機能。
Mozilla Readability による本文自動抽出、GitHub Flavored Markdown（GFM）対応、24項目の細かなカスタマイズが可能。

---

## アーキテクチャ概要

```
[ユーザー操作]
      ↓
[popup.js] 設定読み込み → content script へ変換リクエスト
      ↓
[content.js] DOM処理 → Readability → Turndown → フロントマター付加
      ↓
[popup.js] Markdownプレビュー表示 → コピー / 保存
```

- **Manifest Version:** 3（MV3）
- **最小 Chrome バージョン:** 109
- **Content Script 読み込み方式:** 動的インジェクト（`chrome.scripting.executeScript`）
  - ポップアップを開いた際に未注入であれば自動注入してリトライ

## 機能仕様

### 1. 本文抽出（extractionMode）

| モード | 動作 |
|-------|-----|
| `readability`（デフォルト） | `@mozilla/readability` でメインコンテンツを自動抽出。広告・ナビ等を除いた記事本文を取得 |
| `body` | `document.body.innerHTML` をそのまま変換（ダッシュボード・SPA向け） |

Readability が本文抽出に失敗した場合（`null` 返却）は `document.body.innerHTML` にフォールバックする。

### 2. 選択範囲変換（selectionOnly）

`selectionOnly: true` の場合、`window.getSelection()` で選択中のテキストを `DocumentFragment` として抽出し、Readabilityをバイパスして変換する。
選択がない場合は通常の全文変換にフォールバック。

### 3. 除去オプション

設定画面でON/OFFを切り替え可能。デフォルトはすべてOFF。変換前に `document.cloneNode(true)` したクローンに適用する。

| 設定キー | 対象セレクタ（例） |
|---------|-----------------|
| `removeAds` | `ins.adsbygoogle`, `[class*="ad-"]`, `[class*="advertisement"]`, `[class*="banner"]`, `[data-ad]`, `[class*="sponsor"]`, `[class*="promo"]`, `[class*="affiliate"]` |
| `removeHeader` | `header`, `[role="banner"]`, `[id*="header"]`, `[class*="header"]`, `#site-header`, `.site-header`, `.page-header` |
| `removeFooter` | `footer`, `[role="contentinfo"]`, `[id*="footer"]`, `[class*="footer"]`, `#site-footer`, `.site-footer` |
| `removeNav` | `nav`, `[role="navigation"]`, `[id*="nav"]`, `[class*="nav"]`, `[class*="menu"]`, `.breadcrumb` |
| `removeSidebar` | `aside`, `[role="complementary"]`, `[id*="sidebar"]`, `[class*="sidebar"]`, `.widget-area` |

### 4. HTML → Markdown 変換

TurndownService（headingStyle: `atx`, codeBlockStyle: `fenced`）に GFM プラグインを適用した後、カスタムルールを追加する（GFM より優先）。

| HTML要素 | Markdown変換形式 |
|---------|----------------|
| h1〜h6 | `#`〜`######` |
| `<strong>`, `<b>` | `**テキスト**` |
| `<em>`, `<i>` | `*テキスト*` |
| `<code>`（インライン） | `` `コード` `` |
| `<pre><code>` | 言語指定付き ` ``` ` |
| `<a>` | `[テキスト](URL)` |
| `<img>` | `![alt属性](元URL)` |
| `<table>` | GFM形式 `\| col \| col \|` |
| `<ul>` | `- item` |
| `<ol>` | `1. item` |
| `<blockquote>` | `> テキスト` |
| `<hr>` | `---` |
| `<del>`, `<s>` | `~~テキスト~~`（GFM） |

### 5. 出力オプション

#### 画像の扱い（imageHandling）

| 値 | 動作 |
|----|-----|
| `keep`（デフォルト） | `![alt](url)` 形式で保持 |
| `altOnly` | alt テキストのみ（URL 除去） |
| `remove` | 画像を完全に除去 |

#### リンクの扱い（linkHandling）

| 値 | 動作 |
|----|-----|
| `keep`（デフォルト） | `[テキスト](URL)` 形式で保持 |
| `textOnly` | リンクテキストのみ（URL 除去）。AI プロンプト向けのノイズ削減に有効 |

#### 見出しシフト（headingShift）

| 値 | 動作 |
|----|-----|
| `0`（デフォルト） | シフトなし |
| `1` | h1→h2, h2→h3 ... （他の文書に貼り付ける際の h1 重複防止） |
| `2` | h1→h3, h2→h4 ... |

見出しレベルの上限は h6（`######`）でクランプ。

#### 空行圧縮（collapseBlankLines）

`true` の場合、3行以上の連続空行を2行に圧縮する（`\n{3,}` → `\n\n`）。
トークン数節約・可読性向上に有効。

### 6. フロントマター

Markdown 先頭に付与する YAML メタ情報。`frontmatterEnabled: false` で全体を無効化できる。

```yaml
---
title: "ページタイトル"
url: https://example.com/page
date: 2026-03-19
description: "ページの概要文"
author: "著者名"
tags: ["AI", "research"]
source: 参照先名
status: draft
---
```

| 設定キー | YAMLキー | ソース |
|---------|---------|-------|
| `frontmatterTitle` | `title` | `document.title` |
| `frontmatterUrl` | `url` | `location.href` |
| `frontmatterDate` | `date` | 変換実行日（ISO形式） |
| `frontmatterDescription` | `description` | `meta[name="description"]` / `og:description` |
| `frontmatterAuthor` | `author` | `meta[name="author"]` / `article:author` / `twitter:creator` |
| `frontmatterTags` | `tags` | `frontmatterTagsValue`（カンマ区切り文字列から生成） |
| `frontmatterCustom` | ─ | YAML形式の任意テキスト（複数行可） |

- タイトル内のコロン（`:`）は YAML 解析エラーを防ぐためダブルクォートで囲む
- description / author は取得できなかった場合（空文字列）はフィールド自体を省略する

### 7. ファイル名生成

形式: `{日付}{区切り}{タイトル（最大N文字）}.md`

| 設定キー | デフォルト | 説明 |
|---------|---------|-----|
| `filenameDateFormat` | `YYYY-MM-DD` | `YYYY-MM-DD` / `YYYYMMDD` / `none`（日付なし） |
| `filenameSeparator` | `_` | `_`（アンダースコア）/ `-`（ハイフン） |
| `filenameTitleMaxLength` | `50` | タイトル部の最大文字数（10〜200） |

**タイトルのサニタイズ処理:**
1. ファイル名不正文字（`\ / : * ? " < > |`）を `_` に置換
2. スペースを区切り文字に置換
3. `filenameTitleMaxLength` で切り詰め
4. 末尾の `_` `-` を除去
5. 空になった場合は `untitled` にフォールバック

### 8. 自動コピー（autoCopy）

`true` の場合、ポップアップを開いた直後（変換完了後）に `navigator.clipboard.writeText()` でクリップボードに自動コピーする。
コピー成功でステータス表示、失敗はサイレントに無視。

## chrome.storage.sync スキーマ（24フィールド）

```javascript
{
  // 本文の整え方
  removeAds:               false,  // 広告除去
  removeHeader:            false,  // ヘッダー除去
  removeFooter:            false,  // フッター除去
  removeNav:               false,  // ナビゲーション除去
  removeSidebar:           false,  // サイドバー除去
  extractionMode:          'readability',  // 'readability' | 'body'
  selectionOnly:           false,  // 選択範囲のみ変換

  // 出力形式
  imageHandling:           'keep',    // 'keep' | 'altOnly' | 'remove'
  linkHandling:            'keep',    // 'keep' | 'textOnly'
  headingShift:            0,         // 0 | 1 | 2
  collapseBlankLines:      false,     // 空行圧縮

  // メタ情報（フロントマター）
  frontmatterEnabled:      true,   // フロントマター全体
  frontmatterTitle:        true,   // title フィールド
  frontmatterUrl:          true,   // url フィールド
  frontmatterDate:         true,   // date フィールド
  frontmatterDescription:  false,  // description フィールド
  frontmatterAuthor:       false,  // author フィールド
  frontmatterTags:         false,  // tags フィールド
  frontmatterTagsValue:    '',     // タグ値（カンマ区切り文字列）
  frontmatterCustom:       '',     // カスタムフィールド（YAML複数行）

  // ファイル名・動作
  filenameDateFormat:      'YYYY-MM-DD',  // 'YYYY-MM-DD' | 'YYYYMMDD' | 'none'
  filenameSeparator:       '_',           // '_' | '-'
  filenameTitleMaxLength:  50,            // 10〜200
  autoCopy:                false,         // 変換後に自動コピー
}
```

## UI仕様

### ポップアップ

```
┌───────────────────────────────────────────────────┐
│  [icon] Site2Markdown                         [⚙] │  ← ヘッダー（高さ固定）
├───────────────────────────────────────────────────┤
│  [favicon] example.com                            │  ← ページ情報（ドメイン）
│  記事タイトルがここに表示されます                     │  ← タイトル（2行まで折り返し）
│  /path/to/page                                    │  ← パス（ルートの場合は非表示）
├───────────────────────────────────────────────────┤
│  プロパティ                                     [↓] │  ← 折りたたみ可能
│  title   記事タイトル                              │
│  url     https://example.com/...                  │
│  date    2026-03-20                               │
├───────────────────────────────────────────────────┤
│  Markdown Preview                                 │  ← コンテンツヘッダー
│  ┌─────────────────────────────────────────────┐  │
│  │ ---                                         │  │
│  │ title: "記事タイトル"                         │  │
│  │ url: https://...                            │  │
│  │ date: 2026-03-20                            │  │
│  │ ---                                         │  │
│  │                                             │  │
│  │ # 見出し                                    │  │
│  │ 本文テキスト...                               │  │
│  └─────────────────────────────────────────────┘  │  ← スクロール可能
├───────────────────────────────────────────────────┤
│  [ステータスメッセージ]  [コピー] [保存 ▼]          │  ← アクションバー
└───────────────────────────────────────────────────┘
```

**各エリアの仕様:**

| 要素 | 詳細 |
|------|-----|
| ポップアップサイズ | 幅 400px、高さ 520px（`min-height`） |
| ヘッダー | ロゴ画像（icon48.png, 20px）+ アプリ名 + 設定ボタン |
| ページ情報 | ファビコン（取得できた場合）+ ドメイン / タイトル / パス |
| プロパティ | 折りたたみボタンで `#properties-body` を表示/非表示。`frontmatterEnabled` が false の場合はセクション自体を非表示 |
| Markdownプレビュー | `<textarea readonly>` で変換結果を表示。スクロール可能。変換中は `is-loading` クラスでローディングアニメーション表示 |
| 保存ボタン | スプリットボタン（`btn-split`）構成。メインが「保存」（`saveAs: false`）、トグルで「名前をつけて保存」メニューを表示 |

**ステータス表示:**
- 成功メッセージ（例: 「コピーしました ✓」）: 2.5秒後に自動消去
- エラーメッセージ（例: 「このページでは実行できません」）: 消去されずに表示継続

### 設定画面（⚙ ボタンから開く独立ページ）

サイドバー + コンテンツの2カラムレイアウト。

| タブ | 設定できる項目 |
|------|-------------|
| 本文の整え方 | 抽出モード（Readability / body全体）/ 選択範囲変換 / 除去オプション5種 |
| 出力形式 | 画像の扱い / リンクの扱い / 見出しシフト / 空行圧縮 |
| メタ情報 | フロントマター全体ON/OFF / 含める項目（title, url, date, description, author, tags）/ タグ値 / カスタムフィールド（textarea） |
| ファイル名・動作 | 日付フォーマット / 区切り文字 / タイトル最大文字数 / 自動コピー |
| 出力例 | 現在の設定で生成されるファイル名プレビュー（リアルタイム更新） |

**保存動作:**
- `chrome.storage.sync.set()` を使用
- フッターに「変更を保存」ボタン（変更があるときのみ有効化）
- 保存後にボタンは無効化され「保存済み」ステータスを表示

## パーミッション

| パーミッション | 用途 |
|-------------|-----|
| `activeTab` | 現在のタブ情報（タイトル・URL・ファビコン）取得 |
| `storage` | 設定の永続化（`chrome.storage.sync`） |
| `clipboardWrite` | `navigator.clipboard.writeText()` によるコピー |
| `downloads` | `.md` ファイルのダウンロード保存 |
| `scripting` | Content Script の動的インジェクト |
| `host_permissions: <all_urls>` | 全 URL への Content Script 適用 |

`<all_urls>` は Chrome Web Store 審査で厳しく審査されるパーミッション。ストア掲載説明文でユーザー起点の変換であることを明記して、審査通過リスクを低減する。

## 処理フロー

```
[ユーザーがポップアップを開く]
          ↓
[popup.js] chrome.storage.sync.get() で設定読み込み
          ↓
[popup.js] setPageInfo(tab) でページ情報エリア更新
          ↓
[popup.js] content-wrapper に is-loading クラスを付与（ローディング表示）
          ↓
[popup.js] chrome.tabs.sendMessage({ action: "convert", settings })
          ↓
  ┌── 失敗（Content Script 未注入）──┐
  │                                  ↓
  │     chrome.scripting.executeScript() で動的インジェクト
  │     （Readability.js → turndown.js → turndown-plugin-gfm.js → content.js）
  │                                  ↓
  │     chrome.tabs.sendMessage() 再送
  └──────────────────────────────────┘
          ↓
[content.js] selectionOnly の場合は選択範囲を取得
          ↓
[content.js] document.cloneNode(true) でDOMクローン
          ↓
[content.js] removeUnwantedElements() で不要要素削除
          ↓
[content.js] Readability / body 全体 で本文取得
          ↓
[content.js] TurndownService + GFM + カスタムルールで Markdown 変換
          ↓
[content.js] collapseBlankLines 処理（有効な場合）
          ↓
[content.js] buildFrontmatter() でフロントマター生成・先頭付加
          ↓
[content.js] sendResponse({ markdown, title })
          ↓
[popup.js] is-loading クラスを除去
          ↓
[popup.js] textarea にプレビュー表示・ボタン有効化
          ↓
[popup.js] autoCopy が true なら自動コピー実行

[コピーボタン] → navigator.clipboard.writeText()
[保存ボタン]   → Blob → ObjectURL → chrome.downloads.download({ saveAs: false })
[名前をつけて保存] → chrome.downloads.download({ saveAs: true })
```

## 既知の制限事項

- Readability は記事・ブログ向けの設計のため、SPA・ダッシュボード・ランディングページでは本文抽出精度が低下する場合がある
- 広告除去はクラス名・属性パターンのマッチングに依存するため、サイトによって偽陽性・偽陰性が発生しうる
- `chrome://` など Chrome 内部ページやファイル URL では Content Script が実行できない
- ファビコンは `tab.favIconUrl` が取得できない場合はデフォルト SVG アイコンを表示
- MV3 の CSP 制限により、外部 CDN からのスクリプト読み込みは不可。サードパーティ SDK は vendor/ にバンドル同梱が必要
- `chrome.storage.sync` の上限: キーあたり 8KB、総量 100KB。24フィールドの設定値は通常この範囲内に収まる
